Dead Reckoning

I recently had a need for an absolute positioning system using only dead reckoning. At my disposal was a robot with two motors and encoders on each.

Now, the motors that I used were simple DC motors of questionable quality. While we had built circuits to attempt to drive the two motors at the same speed, physical differences in the motors and circuits prevented this. A naive implementation of dead reckoning assumes that both motors move at the same rate. This is rarely (if ever) the case, and was certainly not the case here. Rather than trying to adjust voltages to match motor speeds, I opted to investigate how to track position with different motor speeds.

Overview

Before diving into the specifics of computing how the robot's position changes over time, I'd like to take a moment to describe how one might implement code for this task. There are several steps, detailed in the list below:

  1. Sample the encoders to track wheel movement
  2. Feed sampled positions into a function to compute changes in position
  3. Apply these changes in position to the global positon of the robot

Let's address (1) first. In this step, we are just sampling how far the wheels move over a given period. This period is defined by some number of encoder counts, perhaps 25. It's also easy to find how far the wheel will move per encoder count, so the distance that the wheel moves is just the number of encoder counts, multiplied by the distance per encoder count.

For (2), we feed in the sampled data from (1) to start calculating the change in the robot's position, and orientation. It is important to track both the position and orientation, because the equations that we will derive for the changes in position are relative to the robot's orientation. One could also say that we need to track the orientation because the changes are in the robot's local space, and cannot be directly applied to the robot's global position. This is not the same for tracking the orientation, or rotation. This can be directly applied to orientation of the robot because the changes in rotation are the same regardless of how the coordinate system is rotated.

Finally, (3) involves updating the robot's position and orientation data. As mentioned in (2), this will require converting the changes in position relative to the robot's orientation into the global space.

I am not going to talk about (1) because it is simple to implement. So let's start with (2). There are two ways in which a robot with two wheels can move. You can run both wheels in the same direction, or you can run the wheels in opposite directions.

Case 1: Linear Motion

Let's look at the case where both motors move in the same direction. If the motors don't move at the same rate, then we can assume that the robot will move in arcs. We assume this based on the additional assumption that the wheels do not slip. So each wheel will travel some (different) distance and the only way for an object to move with these assumptions is by moving in an arc. So, let's draw a digram.

Depcited in the figure is the case of the robot drifting to the right. In the figure we have some variables that need explanation.

Knowns:

  • l1l_1l1​: distance read by the left encoder
  • l2l_2l2​: distance read by the right encoder
  • www: width as measured to the ourside edges of each wheel

Unknowns:

  • θ\thetaθ: the angle of the arc traced out by each wheel
  • r1r_1r1​: radius of the inner arc
  • r2r_2r2​: radius of the outer arc
  • Δx⃗1\Delta \vec x_1Δx1​: change in position of the right (inner) wheel
  • Δx⃗2\Delta \vec x_2Δx2​: change in position of the left (outer) wheel

What we seek is the change in position of the robot itself. We want to track this change in position relative to the robot's pivot point. This is simply the midpoint between both wheels. So the position of the pivot point is just the average of the two wheel positions, or the average of Δx⃗1\Delta\vec x_1Δx1​ and Δx⃗2\Delta\vec x_2Δx2​. This will also apply for Case 2. So we then need to find Δx⃗1\Delta\vec x_1Δx1​ and Δx⃗2\Delta\vec x_2Δx2​, and then compute the average to find the change in the position of the robot.

If we assume θ\thetaθ in radians, we have:

θ=l1r1=l2r2\theta=\frac{l_1}{r_1}=\frac{l_2}{r_2} θ=r1​l1​​=r2​l2​​

Using the diagram above, we also see that r2=w+r1r_2=w+r_1r2​=w+r1​, so we then have:

θ=l1r1=l2w+r1\theta=\frac{l_1}{r_1}=\frac{l_2}{w+r_1} θ=r1​l1​​=w+r1​l2​​

This leaves us with just one unknown, r1r_1r1​, so we can do some algebra this to find its value:

l1(w+r1)=l2r1  ⟹  wl1+l1r1=l2r1  ⟹  r1=wl1l2−l1l_1(w+r_1)=l_2r_1\implies wl_1+l_1r_1=l_2r_1 \implies r_1=\frac{wl_1}{l_2-l_1} l1​(w+r1​)=l2​r1​⟹wl1​+l1​r1​=l2​r1​⟹r1​=l2​−l1​wl1​​

  ⟹  r1=wl2l1−1\implies r_1=\frac{w}{\frac{l_2}{l_1}-1} ⟹r1​=l1​l2​​−1w​

  ⟹  r2=r1+w\implies r_2=r_1+w ⟹r2​=r1​+w

As per described in step (2), we also need to track the orientation. We actually have two equations for θ\thetaθ. We could pick either one for our θ\thetaθ calculation, or average them. I am going to average them, and I am going to assume that the value of θ\thetaθ is this average value for all subsequent calculations.

θ=12(l1r1+l2r2)\theta=\frac{1}{2}(\frac{l_1}{r_1}+\frac{l_2}{r_2}) θ=21​(r1​l1​​+r2​l2​​)

Using these values we can now compute the changes in position of each wheel.

Δx⃗1=r1[1−cos(θ)sin(θ)]\Delta \vec x_1=r_1\begin{bmatrix}{1-cos(\theta)}\\{sin(\theta)}\end{bmatrix} Δx1​=r1​[1−cos(θ)sin(θ)​]

Δx⃗2=r2[1−cos(θ)sin(θ)]\Delta \vec x_2=r_2\begin{bmatrix}{1-cos(\theta)}\\{sin(\theta)}\end{bmatrix} Δx2​=r2​[1−cos(θ)sin(θ)​]

Δx⃗=12(Δx⃗1+Δx⃗2)\Delta \vec x=\frac{1}{2}(\Delta \vec x_1 + \Delta \vec x_2) Δx=21​(Δx1​+Δx2​)

As stated earlier, we want to take the average of Δx⃗1\Delta\vec x_1Δx1​ and Δx⃗2\Delta\vec x_2Δx2​ to compute Δx⃗\Delta\vec xΔx, which is the change in position of the robot's pivot point reltaive to the robot's orientation.

I am using the convension that the first row is the x component, and the second is the y. The 1−cos(θ)1-cos(\theta)1−cos(θ) comes from the fact that cosine starts at 1, while sine starts at 0. Since we want to know by how much the x position changes, we need to take 1 minus cosine.

The equations above are only valid for when the robot is moving forwards, and drifting to the right. If the robot is moving to the left, we need to multiply the x component by -1, since the robot would be drifting in the -x direction. If the robot is moving backwards (and to the right), then we can simply multiply θ\thetaθ by -1 when calculating the changes in position. If the robot is moving backwards, and to the left, we multiply both the x component by -1, and θ\thetaθ by -1. We can determine left and right drift by the magnitudes of l1l_1l1​ and l2l_2l2​. If l1>l2l_1>l_2l1​>l2​ then the robot will drift left. If l1<l2l_1<l_2l1​<l2​, the robot will drift right.

We also need to be carfull when calculating θ\thetaθ. If l1<l2l_1<l_2l1​<l2​ we need to use a different equation for θ\thetaθ:

θ=l2r1=l1r2\theta=\frac{l_2}{r_1}=\frac{l_1}{r_2} θ=r1​l2​​=r2​l1​​

This keeps the assumption that r2r_2r2​ is the outside radius, and r1r_1r1​ is the inside radius.

Case 2: Rotational Motion

In this case, we are assuming that one motor will move in the opposite direction as the other, thus turning the robot in place. In case 1, we already assumed that the wheels of the robot will move in arcs, and the same applies here. Once again, we draw a diagram.

The figure depcits the robot turning to the right in place. We can see that when the speed of the two motors is not the same, the position of the robot's pivot point will actually change when turning in place. The values shown in the figure are the same as in Case 1. As before, we seek the change in position of the robot itself. This will again be the change in position of the robot's pivot point, which will be the average of Δx⃗1\Delta\vec x_1Δx1​ and Δx⃗2\Delta\vec x_2Δx2​.

As before we have θ\thetaθ:

θ=l1r1=l2r2\theta=\frac{l_1}{r_1}=\frac{l_2}{r_2} θ=r1​l1​​=r2​l2​​

From the diagram we see that r2=w−r1r_2=w-r_1r2​=w−r1​, and substituing in we have:

l1r1=l2w−r1  ⟹  l1(w−r1)=r1l2  ⟹  l1w−l1r1=r1l2  ⟹  r1=l1wl1+l2\frac{l_1}{r_1}=\frac{l_2}{w-r_1}\implies l_1(w-r_1)=r_1l_2\implies l_1w-l_1r_1=r_1l_2\implies r_1=\frac{l_1w}{l_1+l_2} r1​l1​​=w−r1​l2​​⟹l1​(w−r1​)=r1​l2​⟹l1​w−l1​r1​=r1​l2​⟹r1​=l1​+l2​l1​w​

  ⟹  r1=wl2l1+1\implies r_1=\frac{w}{\frac{l_2}{l_1}+1} ⟹r1​=l1​l2​​+1w​

  ⟹  r2=w−r1\implies r_2=w-r_1 ⟹r2​=w−r1​

And as before we also have Δθ\Delta\thetaΔθ:

θ=12(l1r1+l2r2)\theta=\frac{1}{2}(\frac{l_1}{r_1}+\frac{l_2}{r_2}) θ=21​(r1​l1​​+r2​l2​​)

As with case 1, we now have all the information we need to solve for Δx1\Delta x_1Δx1​ and Δx2\Delta x_2Δx2​.

Δx⃗1=r1[cos(θ)−1−sin(θ)]\Delta\vec x_1=r_1\begin{bmatrix}{cos(\theta)-1}\\{-sin(\theta)}\end{bmatrix} Δx1​=r1​[cos(θ)−1−sin(θ)​]

Δx⃗2=r2[1−cos(θ)sin(θ)]\Delta\vec x_2=r_2\begin{bmatrix}{1-cos(\theta)}\\{sin(\theta)}\end{bmatrix} Δx2​=r2​[1−cos(θ)sin(θ)​]

Δx⃗=12(Δx⃗1+Δx⃗2)\Delta\vec x=\frac{1}{2}(\Delta\vec x_1+\Delta\vec x_2) Δx=21​(Δx1​+Δx2​)

As with case one, we use sines and cosines to solve for the change in position for each wheel. In this case, we see that the rigth wheel moves in the negative x and y direction, so we just multiple Δx⃗1\Delta\vec x_1Δx1​ by -1. We also have Δx⃗\Delta\vec xΔx as the average in the change in positions of each wheel, as before.

Again, these equations are only valid when the robot is turning right, and l2>l1l_2>l_1l2​>l1​. If the robot is turning left, we need to multiply θ\thetaθ by -1 when calculating the changes in position. And as with Case 1, if l2<l1l_2<l_1l2​<l1​ we use the alternate equation for calculating θ\thetaθ. This allows us to reduce the number of cases we have to deal with. For example, we could have the case that l2>l1l_2>l_1l2​>l1​, but we are turning to the left, which would imply negative movement in y, but positive in x. This would throw some more negatives into the works if dealt with using if/else, but using an alternative equation for θ\thetaθ cleans this up, while maintaining all the proper signs in all cases.

Applying Changes

Now that we have derived formulas for the local changes in the robot's position, we need to convert these to changes in global position.

Let's say we have a value θ0\theta_0θ0​ that tracks the robot's orientation. To transform our values, we can simply use a rotation matrix, with θ0\theta_0θ0​ as its parameter. So let's express the global change in position as Δx⃗g\Delta\vec x_gΔxg​:

Δx⃗g=[cos(θ0)−sin(θ0)sin(θ0)cos(θ0)]Δx⃗\Delta\vec x_g=\begin{bmatrix}{cos(\theta_0)}&{-sin(\theta_0)}\\{sin(\theta_0)}&{cos(\theta_0)}\end{bmatrix}\Delta\vec x Δxg​=[cos(θ0​)sin(θ0​)​−sin(θ0​)cos(θ0​)​]Δx

This applies to both Cases 1 and 2. Once we have computed this value, we can finally add this to whatever vector is tracking the robot's global position.

To update the robot's rotation, all we need to do is add θ\thetaθ to our value θ0\theta_0θ0​ after we have computed our global change. By convention, however, turning right means that ∣θ∣|\theta|∣θ∣ should be subtracted, while turning left has ∣θ∣|\theta|∣θ∣ added.
